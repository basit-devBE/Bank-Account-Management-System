This repository follows a structured Git workflow to ensure efficient collaboration and code management. Below are the key practices and guidelines to follow


The repository uses a feature-branch workflow.
master: branch is reserved for stable releases.
refactor: branch used for restructuring  and improving the existing codebase
feature/testing: branch used for adding new testing features and improvements.
feature/exceptions: branch used for implementing exception handling features.

## Branch Naming Conventions
- Use descriptive names for branches that reflect the purpose of the branch.
- Use hyphens to separate words (e.g., feature/add-login, bugfix/fix-something).
